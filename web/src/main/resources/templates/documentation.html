E3 Request Schema

Type: object

Schema description for the JSON request file that is accepted by E3.

root analysisObject
Type: object

Defines general analysis request variables.

root analysisObject type
Type: array of enum (of string)

Analysis type.

Each item of this array must be:

root analysisObject type type items
Type: enum (of string)

Must be one of:

  • "LCCA"
  • "BCA"
  • "COST_LOSS"
  • "PROFIT_MAXIMIZATION"
  • "OTHER"

root analysisObject projectType
Type: array of enum (of string)

Project category type.

Each item of this array must be:

root analysisObject projectType projectType items
Type: enum (of string)

Must be one of:

  • "BUILDING"
  • "INFRASTRUCTURE"
  • "RESILIENCE"
  • "MANUFACTURING"
  • "OTHER"

root analysisObject outputObjects
Type: array of enum (of string)

Objects to report after the analysis is complete. Some objects may take longer than others to create

Each item of this array must be:

root analysisObject outputObjects outputObjects items
Type: enum (of string)

Must be one of:

  • "REQUIRED"
  • "OPTIONAL"
  • "MEASURES"
  • "SENSITIVITY"
  • "UNCERTAINTY"

root analysisObject studyPeriod
Type: integer

Amount of time the analysis will be covering. The initial time-step (0) is included automatically, e.g. a study period of 10 will report 11 values, time-steps 0-10.

root analysisObject timestepValue
Type: enum (of string) Default: "YEAR"

Defines what the time-steps of this analysis are. NOTE: currently no values besides "YEAR" are implemented.

Must be one of:

  • "YEAR"
  • "QUARTER"
  • "MONTH"
  • "DAY"

root analysisObject timestepComp
Type: enum (of string) Default: "END_OF_YEAR"

Defines when the discounting will be applied on monetary values.

Must be one of:

  • "END_OF_YEAR"
  • "MID_YEAR"
  • "CONTINUOUS"

root alternativeObjects
Type: array of object

Array of JSON alternative object descriptions that group BCNs into alternatives.

Must contain a minimum of 1 items

Each item of this array must be:

root alternativeObjects alternativeObjects items
Type: object

root alternativeObjects alternativeObjects items id
Type: integer

Integer ID of this alternative.

Value must be greater or equal to 0

root alternativeObjects alternativeObjects items name
Type: string

Name of this alternative.

root alternativeObjects alternativeObjects items bcns
Type: array of integer

Array of BCN IDs that are included in the alternative. BCN IDs can be shared with other alternatives.

Must contain a minimum of 1 items

All items must be unique

Each item of this array must be:

root alternativeObjects alternativeObjects items bcns bcns items
Type: integer

Value must be greater or equal to 0

root bcnObjects
Type: array of object

Array of JSON BCN object descriptions. Must be the same as the number defined in analysis.

Each item of this array must be:

root bcnObjects bcnObjects items
Type: object

root bcnObjects bcnObjects items id
Type: integer

Integer ID of this BCN.

Value must be greater or equal to 0

root bcnObjects bcnObjects items altIds

List of alternatives this BCN is a part of.

  • Option 1
  • Option 2
root bcnObjects bcnObjects items altIds oneOf item 0
Type: integer

Value must be greater or equal to 0

root bcnObjects bcnObjects items altIds oneOf item 1
Type: array of integer

Must contain a minimum of 1 items

All items must be unique

Each item of this array must be:

root bcnObjects bcnObjects items altIds oneOf item 1 item 1 items
Type: integer

root bcnObjects bcnObjects items residualValue
Type: boolean Default: false

Determines if residual value will be calculated and added to the end of this BCN.

Type: boolean Default: false

Determines if this BCN is a manually calculated residual value that does not include any other BNC values.

root bcnObjects bcnObjects items life
Type: integer Default: 0

Lifetime of this BCN. Can be greater than the study period. TODO: add constraint on default value if recur is defined

Value must be greater or equal to 0

root bcnObjects bcnObjects items recur
Type: object Default: null

Defines the options for when a BCN is applied multiple times throughout the study period.

root bcnObjects bcnObjects items recur interval
Type: integer

Number of time-steps between each recurrence of this BCN. i.e. interval of 1 means BCN is applied for every time step, interval of 2 means BCN is applied for every OTHER time-step, etc.

Value must be greater or equal to 1

root bcnObjects bcnObjects items recur end
Type: number

The end time-step of the recurrence. Can be after the study period. TODO: default is study period

root bcnObjects bcnObjects items quantity
Type: number

Number of non-monetary assets included in this BCN. e.g. for a BCN which defines an energy consumption of 100MW, the quantity would be 100.

Value must be greater or equal to 0

root bcnObjects bcnObjects items quantityValue
Type: number

Monetary value of the BCN asset. e.g. for a BCN which defines an energy consumption of 100MW at $0.05/MW, quantityValue would be 0.05 which would generate a total value in the analysis as 100MW × $0.05/MW = $5

root bcnObjects bcnObjects items quantityVarRate
Type: enum (of string) Default: null

Determines how quantities are modified by their var values. TODO: define differences

Must be one of:

  • "PERCENT_DELTA"
  • "YEAR_BY_YEAR"

root bcnObjects bcnObjects items quantityVarValue
Default: null

Values to alter quantity throughout the study period. Must be equal to the study period even if the BCN does not last through the study period. TODO: add constraint on size with study period

  • Option 1
  • Option 2
root bcnObjects bcnObjects items quantityVarValue oneOf item 0
Type: number
root bcnObjects bcnObjects items quantityVarValue oneOf item 1
Type: array of number

Each item of this array must be:

root bcnObjects bcnObjects items quantityVarValue oneOf item 1 item 1 items
Type: number

root bcnObjects bcnObjects items quantityUnit
Type: string Default: null

Unit of BCN which is used to group together similar BCNs for reporting purposes. e.g. for a BCN defining an energy consumption of 100MW, unit would be "MW".

root sensitivityObjects
Type: array of object

Defines desired sensitivity analyses.

Each item of this array must be:

root sensitivityObjects sensitivityObjects items
Type: object

root sensitivityObjects sensitivityObjects items id
Type: integer

ID of this sensitivity analysis. Must be non-negative.

Value must be greater or equal to 0

root sensitivityObjects sensitivityObjects items altId
Type: integer

ID of the alternative that this sensitivity analysis is associated with.

Value must be greater or equal to 0

root sensitivityObjects sensitivityObjects items variable
Type: string

Path of the variable to alter for sensitivity. Dot-notation or bracket notation are valid. For example, altering the quantityValue in BCN 1 could be "bcnObjects.1.quantityValue" or "bcnObjects[1].quantityValue".

root sensitivityObjects sensitivityObjects items diffType
Type: enum (of string)

Defines how to alter the specified variable.

Must be one of:

  • "PERCENT"
  • "POSITIVE_PERCENT"
  • "NEGATIVE_PERCENT"
  • "GROSS"

root sensitivityObjects sensitivityObjects items diffValue
Type: number

The amount of change to pass into the specified alter function.
